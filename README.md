<h2 id="whatsbrightbi">What's BrightBI?</h2>

*A business intelligence tool to analyze enterprise data and to share reports 
anytime anywhere on any devices.*

<h3 id="tool">A business intelligence tool</h3>

  BrightBI is a BI(Business Intelligence) tool. Users can load any source data 
  into BrightBI, clean and refine the data, then analyze the data to create multiple 
  dynamic pivot tables and pivot charts. Users can also combine these pivot tables 
  and pivot charts to create any kind of reports and share all these analytics and 
  reports with others.
  
  BrightBI is lightweight. By leverage modern web 2.0 technologies, BrightBI put 
  all BI operations into an unify intuitive web user interface.
  The very easy installation and setup process make it particularly suit for medium 
  and small businesses to create a simple but powerful data analytics and reporting system.
  
<h3 id="analyzedata">To analyze enterprise data</h3>

  BrightBI provides dynamically and intuitively data analytics to everybody within enterprise.
  Enterprise managers, especially executive managers, need not waiting for weekly 
  or monthly static reports to make their decisions any more. They can do all 
  kinds of multiple dimensional analysis right away by themselves anytime with 
  intuitive drag and drops.

<h3 id="sharereport">And to share reports</h3>

  BrightBI also provides a reporting system to everybody within enterprise.
  Users can save their analytics results as reports anytime. Reports can be saved
  as files or be saved into database and be opened by web links anytime later on.

  Sharing reports to others can be achieved within system and any authorized 
  report receiver can reopen the report by follow a web link to do further analysis.
  Users can also make comments to any report so as to communicate with others.

<h3 id="anytimewhere">Anytime anywhere on any devices</h3>

  BrightBI is pure web based. The only requirement to use BrightBI is a modern internet browser.
  All user interfaces are based on latest HTML5 and javascript technology and can
  be accessed by users any time anywhere on any devices.

<h2 id="whybrightbi">Why BrightBI?</h2>

* Simple
* Powerful
* Low cost

<h3 id="simple">Simple</h3>

*Simpel is the best!*

[Installation](#installation) and [setup](#setup) of BrightBI are all quite simple. 
Just follow several steps, everybody can finish it within half an hour and even 
without consulting profession BI experts or advanced IT professionals. This is a 
great news for medium and small size businesses who can't afford hiring BI or 
IT professionals.

Using of BrightBI is also quite simple. All user interfaces are very intuitive 
so that users will have little learn curves.  BrightBI targets to expand dynamically 
data analytics and reporting from special enterprise BI department to everybody 
within enterprise, especially high level managers. By using BrightBI, BI is not primarily
the domain of a small band of analysts within large companies any more, it can 
be rapidly spread throughout whole organization.

<h3 id="powerful">Powerful</h3>

*Powerful is the key!*

* **Widely data sources support**

    * Data file as data source

      BrightBI supports to load and analyze all kinds of enterprise data. It provides 
      an unify user interface for users to load their source data from data files into system, 
      no matter it is in CSV format, Microsoft Excel format, or Google Spreadsheet format, 
      and no matter it locates on users local drive, or cloud drives, or any web server 
      as an internet link.

      BrightBI will identify the loaded data automatically so that the data can be used to
      do further analysis directly, it also provides data cleaning and refining  
      tools for user to clean and reorganize the source data.
    
    * OLAP Cube as data source

      BrightBI support access [Online analytics process (OLAP)] [OLAP] cubes directly
      through [XML for Analysis (XMLA)] [XMLA] protocol. XMLA is a Simple Object 
      Access Protocol (SOAP)-based XML protocol, designed specifically for universal 
      data access to any standard multidimensional data source residing on the Web.

      Almost all major BI providers support this industry standard so that BrightBI can 
      easily access their OLAP engines, like Microsoft Analytics Service, Mondrian 
      open source OLAP server, Oracle Essbase, IBM DB2 OLAP server, SAS OLAP server, 
      SAP OLAP server, etc.

      This is a great news for big companies who invested on creating OLAP cubes 
      already. BrightBI can help these companies to easily expand the use of OLAP cubes to
      a wider community of users, especially enterprise executive managers.

    * SQL database and big data database as data source

      BrightBI provides a lots of APIs to load source data from backend database system. 
      This includes all kinds of SQL relational databases and big data databases.

   [OLAP]: http://en.wikipedia.org/wiki/Online_analytical_processing
   [XMLA]: http://en.wikipedia.org/wiki/XML_for_Analysis

* **Full functional data analytics**

    * Creating Excel-like pivot table and pivot chart in minutes

      The [pivot table] [pivottable] is a very useful tool for the data analyst. 
      There are a lot of pivot table implementations in the market.  Among all 
      the implementations, Microsoft Excel is definitely the best one. Everyday, 
      millions of users use it to analyze their data, to create their reports. 
      
      BrightBI looks like a web version Excel. It implements almost all functionalities 
      of Excel Pivot Table but with more intuitive user interface. What you can do 
      with Excel pivot table, you can do the same in BrightBI.
      
    * Creating multiple pivot tables  at same time
       
      BrightBI supports to create multiple pivot tables and pivot charts at same time. 
      Each pivot tables and charts can be customized independently and they can based 
      on different data sources. 
      
    * Show pivot table view and pivot chart view  together 
    
      BrightBI supports to show pivot table and pivot chart together, user can also switch 
      views easily and all actions on one view will automatically apply to other views.
      
    * Easily pivot operations with cutomized summarize functions and data formats
    
      BrightBI supports customize all spects of pivot operation. Changing summarize functions, 
      changing sorts and filters, changes data formats are very simple and easy. And user 
      can see these changes immidiately.
      
    * Powerful calculated fields
    
      BrightBI supports to add any calculated fields and the formula of calculated fields
      can be any combination of existing fields, constants and math functions. All valid
      javascript expression can be used as fomula and even if-else logic can be used 
      through iif like function.
      
    * Powerful cutomized options for pivot table and pivot chart
    
      BrightBI supports to customize almost all aspects of pivot table and pivot chart. 
      For example: table size, table theme, table layout, totals and subtotals, chart size,
      chart types, chart themes, chart legends, tooltips, etc. All these changes are very 
      intuitive and user can see these changes immidiately.

      
  [pivottable]: http://en.wikipedia.org/wiki/Pivot_table  
   
      
* **Widely reporting support**
    
    * Create  multiple reports at the same time
    
      BrightBI supports to create mutiple reports based on all available pivot tables and pivot charts.
      User can cutomize these reports independently. And all piovt operations on pivot tables and 
      pivot charts will apply to all reports which are based on these tables and charts.

    * Support all kinds of report export formats 
    
      BrightBI supports to export any reports to PDF, HTML, PNG, JPEG, GIF formats. 
      Each pivot table or pivot chart can also be exports to these format independently.
      
    * Support to save reports both on local files or save into backend databases.  

* **Sharing**
 
    * Sharing whole data analytics 
   
      Users can save all their source data and their data analytics into a WPT 
      format file anytime. It can be saved to local drive or saved into
      backend database directly. Users can easily reopen this file by loading from 
      local drive or loading from database anytime. Everyting will be exactly the same 
      as when the file saved.

      There is a dashboard for each user after they logged in. The dashboard of this 
      user will show all the WPT files saved into database which he/she has been 
      authorized to use. By click any WPT file link, that file will opened in WPT.

    * Sharing reports 
    
      Users can export any report to a file independently. The supporting export 
      file formats are PDF/HTML/PNG/JPEG/GIF. Report file can also be saved into 
      backend database directly. And by click a link in dashboard, they will be 
      download from server.

    * Sharing through email or assign 
    
      Users can also share WPT format file/link or report file/link through email 
      to outside or assign to another users within WPT.
      
    * Comment service on any WPT or report
    
      Users can add any comments to a paricular data analytics or reports. Comment
      service is a great tool for users to comminicate with each other.

<h2 id="howbrightbi">How to use BrightBI?</h2>

BrightBI can be used as an independent enterprise BI solution, it can be also integrated into
any other web application or web site. OEM or SaaS integrations are also supported.

<h3 id="independent">Use as an independent enterprise business intelligence solution</h3>

* Install open source [Node.js](http://nodejs.org) platform on your server machine.
  Run "node --version" to check whether Node.js has been installed correctly.
* Install open source [MongoDB](http://www.mongodb.org/) NoSQL database on the same server machine or another server machine in the same network.
  After MongoDB installation, start MongoDB follow the [MongoDB document](http://docs.mongodb.org/manual/)
  If you don't want install MongoDB, you can try use cloud based MongoDB. There are lots of cloud based MongoDB in the market, for example, [Compose (original MongoHQ)](https://www.compose.io/) and [MongoLab](https://mongolab.com/).
* Download [WPT](http://webpivottable.com/download) and unzip it to a directory.
  If MongoDB is not on the same server machine, open "app/config.js" file to change "db" parameter to point to correct MongoDB URL.
* Go to unzipped directory, run "npm install"
* run "node app"
* Open any browser, access http://localhost:8002

<h3 id="integration">Integrate into any web application as javascript component </h3>

* Download [WPT](http://webpivottable.com/download) and unzip it to a directory.
* Copy all contents under public subdirectory into your web application www root directory.
